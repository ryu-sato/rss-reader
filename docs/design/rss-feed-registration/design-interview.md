# RSSシードの登録 設計ヒアリング記録

**作成日**: 2026-03-13
**ヒアリング実施**: step4 既存情報ベースの差分ヒアリング

## ヒアリング目的

新規プロジェクトのため既存設計なし。RSSフィード登録機能の技術設計を一から決定するためのヒアリングを実施。

**質問日時**: 2026-03-13
**カテゴリ**: 技術選択
**背景**: DBアクセス方法によりコード構造とマイグレーション管理が変わるため

- データアクセス層の設計が 🔵 確定

## ヒアリング結果サマリー

### 確認できた事項
- Prisma ORM を使用（マイグレーション管理含む）
- Route Handlers で REST API 実装
- rss-parser でRSS/Atom パース
- shadcn/ui + Tailwind CSS でUI構築
- SSRF対策実施（プライベートIPブロック）
- エラーメッセージは英語
- description/memo 最大1000文字

### 設計方針の決定事項
- アーキテクチャ: Next.js フルスタックモノリス（App Router）
- APIスタイル: REST（Route Handlers）
- 認証: なし
- DBスキーマ: Prisma schema.prisma で管理
- SSRFガード: DNS解決後にIPアドレスを検証

### 残課題
- 特になし（🔴赤信号は全てヒアリングで解消）

### 信頼性レベル分布

**ヒアリング前（要件定義段階）**:
- 🔵 青信号: 4件（技術スタックの基本項目）
- 🟡 黄信号: 8件
- 🔴 赤信号: 4件（SSRF対策、エラー言語、文字数制限等）

**ヒアリング後**:
- 🔵 青信号: 16件 (+12)
- 🟡 黄信号: 5件 (-3)
- 🔴 赤信号: 0件 (-4)
