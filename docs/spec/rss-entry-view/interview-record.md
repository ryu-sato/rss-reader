# RSSエントリー閲覧 ヒアリング記録

**作成日**: 2026-03-14
**ヒアリング実施**: step4 既存情報ベースの差分ヒアリング

## ヒアリング目的

既存の rss-feed-registration 設計文書・実装を確認し、RSSエントリー閲覧機能に関する不明点や曖昧な部分を明確化するためのヒアリングを実施しました。

## 質問と回答

### Q1: 閲覧方式

**カテゴリ**: 未定義部分詳細化
**背景**: 既存実装にはフィード管理のみがあり、エントリー閲覧方式が未定義だったため

**質問**: RSSエントリー閲覧の基本的な動作を教えてください。ユーザーはどのようにエントリーを閲覧しますか？

**回答**: 全フィードまとめ読み

**信頼性への影響**:
- REQ-001（全フィードまとめ表示）の信頼性が 🔴 → 🔵 に向上

---

**質問**: エントリーの取得タイミングをどうしますか？

**回答**: DBキャッシュあり

**信頼性への影響**:

### Q3: 定期自動取得間隔

**カテゴリ**: 未定義部分詳細化
**背景**: DBキャッシュありの場合の更新方法が未定義だったため

**カテゴリ**: 未定義部分詳細化
**背景**: ページネーションの有無・件数が未定義だったため

**質問**: 1ページに表示する記事件数とソート順を教えてください / 記事一覧のページネーションは必要ですか？

### Q6: 記事クリック時の動作

**カテゴリ**: 未定義部分詳細化
**背景**: 記事クリック後の動作（外部リンク・アプリ内閲覧）が未定義だったため

**質問**: 記事をクリックしたときの動作を教えてください

**回答**: エントリーの全ての情報をモーダル表示して、後でメタ情報を付加できるようにする。メタ情報には「後でよむ」「タグ」が設定できる。

**信頼性への影響**:
- REQ-006（モーダル表示）、REQ-007（前後ナビ）の信頼性が 🔴 → 🔵 に向上
- REQ-009（後で読む）、REQ-010（タグ）の信頼性が 🔴 → 🔵 に向上

## ヒアリング結果サマリー

### 確認できた事項
- 閲覧方式: 全フィードまとめ読み（フィード選択型ではない）
- エントリー取得: DBキャッシュあり + 1時間ごと定期自動取得
- 一覧表示: タイトル・フィード名、20件/ページ、公開日時降順
- モーダル: 全文・元記事リンク・メタ情報編集・前後ナビ
- 既読管理: あり
- タグ: 自由入力作成 + 既存タグ選択
- 全文DB保存: あり
- 保存上限: 1フィードあたり500件
- フィルター: フィード別・タグ別両方

### 追加/変更要件
- 全文DB保存（当初メタのみと想定していたが全文に変更）
- モーダルに前後ナビ追加（当初想定外）
- フィード別・タグ別フィルター（当初想定外）
- 定期自動取得（手動更新ではなく自動化）

### 残課題
- EDGE-102: タグ名の大文字小文字の扱い（同一タグとして扱うか否か）
- EDGE-103: フィード・タグ複数選択時のフィルター動作（AND/OR）

### 信頼性レベル分布

**ヒアリング前**:
- 🔵 青信号: 0件
- 🟡 黄信号: 5件
- 🔴 赤信号: 20件以上

**ヒアリング後**:
- 🔵 青信号: 20件 (+20)
- 🟡 黄信号: 10件 (+5)
- 🔴 赤信号: 2件 (-18以上)
