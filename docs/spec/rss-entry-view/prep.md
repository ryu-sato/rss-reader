**【信頼性レベル凡例】**:
- 🔵 **青信号**: 要件定義書・設計文書・ユーザヒアリングで明確に必要と判明したタスク
- 🟡 **黄信号**: 要件定義書・設計文書から妥当に推測されるタスク

## 必須（実装開始前に完了が必要）

以下のタスクが完了していないと、実装フェーズでブロッカーになります。

- [ ] **定期取得方式の決定** 🔵 *REQ-211: 1時間ごと定期自動取得より*
  - Next.js App Router では標準のcronジョブ機能がないため、以下のいずれかを選択：
    1. **node-cron**（Node.jsプロセス内スケジューラー）: `pnpm add node-cron`、サーバー起動時に自動実行
    2. **Vercel Cron Jobs**（Vercelデプロイ時のみ）: `vercel.json` に設定
    3. **外部cronサービス**（cron-job.orgなど）: APIエンドポイントを外部から定期呼び出し
    4. **OS/Dockerのcron**: devcontainer環境でcrontabを設定
  - 開発環境（devcontainer）での動作も考慮した選択が必要
  - 関連要件: REQ-211

## 推奨（実装中に用意できればOK）

実装を開始できますが、該当機能の実装前までに準備してください。

- [ ] **HTMLサニタイズライブラリの選定** 🟡 *NFR-101: XSS対策より*
  - エントリーのHTML全文をモーダルに表示する際のXSS対策用ライブラリを選定
  - 候補: `dompurify`（ブラウザ向け）、`sanitize-html`（サーバー向け）
  - Next.js の Server/Client Component の分離を考慮して選択
  - 関連要件: NFR-101

## 確認事項（判断が必要）

実装方針に影響するため、早めの判断・確認が推奨されます。

- [ ] **タグ名の大文字小文字の扱い** 🔴 *EDGE-102: 推測（要確認）*
  - 「技術」と「技術」は同一タグか、別タグか
  - 英語タグの場合: 「Tech」と「tech」は同一か
  - 推奨: 大文字小文字を区別しない（case-insensitive）の方がユーザビリティが高い
  - 関連要件: EDGE-102

- [ ] **フィード・タグ複数選択フィルターの動作（AND/OR）** 🔴 *EDGE-103: 推測（要確認）*
  - フィードAとタグ「技術」を同時に選択した場合の動作を決定
  - 選択肢: AND（フィードA かつ タグ技術）or OR（フィードA または タグ技術）
  - 推奨: AND条件（より直感的な絞り込み）
  - 関連要件: EDGE-103
