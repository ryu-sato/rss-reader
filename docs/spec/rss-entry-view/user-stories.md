## エピック1: エントリー一覧閲覧

### ストーリー 1.1: 全フィードのエントリーをまとめて読む 🔵

**信頼性**: 🔵 *ヒアリングQ1: 全フィードまとめ読みより*

**私は** アプリユーザー **として**
**全フィードの最新記事を1箇所でまとめて確認したい**
**そうすることで** 複数フィードを個別に確認する手間なく効率的に情報収集できる

**関連要件**: REQ-001, REQ-002, REQ-003

**詳細シナリオ**:
1. ホームページ（またはエントリー一覧ページ）を開く
2. 全フィードのエントリーが公開日時降順で表示される
3. 各エントリーにはタイトルとフィード名が表示される
4. 20件ずつページネーションで表示される

**前提条件**:
- 1件以上のRSSフィードが登録済みである
- DBにエントリーが1件以上キャッシュされている

**優先度**: Must Have

---

### ストーリー 1.2: エントリーをフィード別に絞り込む 🔵

**信頼性**: 🔵 *ヒアリングQ11: フィルター（両方）より*

**私は** アプリユーザー **として**
**特定のフィードの記事だけを見たい**
**そうすることで** 興味のある情報源に集中して読める

**関連要件**: REQ-004

**詳細シナリオ**:
1. エントリー一覧のフィルターからフィードを選択する
2. 選択したフィードのエントリーのみが表示される
3. フィルターを解除すると全フィードの表示に戻る

**前提条件**:
- 複数のRSSフィードが登録済みである

**優先度**: Must Have

---

### ストーリー 1.3: エントリーをタグ別に絞り込む 🔵

**信頼性**: 🔵 *ヒアリングQ11: フィルター（両方）より*

**私は** アプリユーザー **として**
**特定のタグが付いた記事だけを見たい**
**そうすることで** 自分が重要視した記事をあとから効率的に見返せる

**関連要件**: REQ-005

**詳細シナリオ**:
1. エントリー一覧のフィルターからタグを選択する
2. 選択したタグが付いているエントリーのみが表示される
3. フィルターを解除すると全エントリーの表示に戻る

**前提条件**:
- 1件以上のタグが作成済みであり、エントリーに付与されている

**優先度**: Must Have

## エピック2: エントリー詳細閲覧

### ストーリー 2.1: モーダルでエントリーを詳細閲覧する 🔵

**信頼性**: 🔵 *ヒアリングQ6: モーダル表示より*

**私は** アプリユーザー **として**
**記事をクリックしてアプリを離れずに本文を読みたい**
**そうすることで** 読書体験を中断せず効率よく記事を消費できる

**関連要件**: REQ-006, REQ-101

**詳細シナリオ**:
1. エントリー一覧でエントリーをクリックする
2. モーダルが開き、タイトル・公開日時・全文が表示される
3. 元記事へのリンクボタンが表示される
4. モーダルを開くと自動的に既読状態になる

**前提条件**:
- エントリーがDBにキャッシュされている

**優先度**: Must Have

---

### ストーリー 2.2: モーダルで前後の記事に移動する 🔵

**信頼性**: 🔵 *ヒアリングQ6: 前後モーダル移動ボタンより*

**私は** アプリユーザー **として**
**モーダルを閉じずに次の記事・前の記事に移動したい**
**そうすることで** 連続して複数の記事を素早く読める

**関連要件**: REQ-007

**詳細シナリオ**:
1. モーダルを開いた状態で「次の記事」ボタンをクリックする
2. 現在のソート順・フィルター条件に従った次のエントリーが表示される
3. 「前の記事」ボタンで前のエントリーに戻れる
4. 最初・最後のエントリーでは対応するボタンが無効化される

**前提条件**:
- エントリーが2件以上存在する

**優先度**: Must Have

## エピック3: メタ情報管理

### ストーリー 3.1: エントリーを既読にする 🔵

**信頼性**: 🔵 *ヒアリングQ7: 既読管理 必要より*

**私は** アプリユーザー **として**
**読んだ記事を既読としてマークしたい**
**そうすることで** まだ読んでいない記事と読んだ記事を区別できる

**関連要件**: REQ-008, REQ-101

**詳細シナリオ**:
1. モーダルでエントリーを開くと自動的に既読になる
2. 既読エントリーは一覧で視覚的に区別される（例: 薄い表示）
3. モーダルから手動で既読・未読を切り替えられる

**前提条件**:
- エントリーが存在する

**優先度**: Must Have

---

### ストーリー 3.2: 記事に「後で読む」を設定する 🔵

**信頼性**: 🔵 *ヒアリングQ6: 後でよむより*

**私は** アプリユーザー **として**
**後で読みたい記事にフラグを立てたい**
**そうすることで** 後でまとめて読むべき記事にすぐアクセスできる

**関連要件**: REQ-009

**詳細シナリオ**:
1. モーダルで「後で読む」ボタンをクリックする
2. エントリーに「後で読む」フラグが付与される
3. 再クリックでフラグが解除される
4. 「後で読む」フィルターで対象エントリーを絞り込める

**前提条件**:
- エントリーのモーダルが開いている

**優先度**: Must Have

---

### ストーリー 3.3: 記事にタグを付ける 🔵

**信頼性**: 🔵 *ヒアリングQ6,Q8: タグ機能より*

**私は** アプリユーザー **として**
**記事に自分でタグを付けて分類したい**
**そうすることで** 後でタグで絞り込んで関連記事をまとめて見られる

**関連要件**: REQ-010, REQ-011, REQ-012

**詳細シナリオ**:
1. モーダルのタグ入力欄に新しいタグ名を入力してEnterまたは選択する
2. 入力したタグが新規作成されてエントリーに付与される
3. 既登録タグの場合は候補から選択して付与できる
4. タグを×ボタンで削除できる

**前提条件**:
- エントリーのモーダルが開いている

**優先度**: Must Have

## エピック4: 自動更新

### ストーリー 4.1: エントリーが自動的に更新される 🔵

**信頼性**: 🔵 *ヒアリングQ3: 定期自動取得・1時間ごとより*

**私は** アプリユーザー **として**
**特に操作しなくてもRSSの最新記事が自動で取り込まれてほしい**
**そうすることで** 毎回手動で更新する手間なく常に最新情報を確認できる

**関連要件**: REQ-211, REQ-402, REQ-103

**詳細シナリオ**:
1. バックグラウンドで1時間ごとに全フィードのRSSを取得する
2. 新しいエントリーがあればDBに保存される
3. 1フィードあたり500件を超えた場合、古いものから削除される
4. 取得失敗のフィードはスキップして他フィードの処理を継続する

**前提条件**:
- 1件以上のRSSフィードが登録済みである

**優先度**: Must Have
