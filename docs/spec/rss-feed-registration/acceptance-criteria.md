## REQ-001: URLを入力するフォームの提供 🔵

**信頼性**: 🔵 *ヒアリングQ1より*

### Given（前提条件）
- アプリのフィード管理ページが表示されている

### When（実行条件）
- ページを表示する

### Then（期待結果）
- RSS URLを入力できるテキストフィールドが表示される
- 「登録」ボタンが表示される

### テストケース

#### 正常系

- [ ] **TC-001-01**: URLフォームの表示 🔵
  - **入力**: フィード管理ページにアクセス
  - **期待結果**: URLテキストフィールドと登録ボタンが表示される
  - **信頼性**: 🔵 *ヒアリングQ1より*

## REQ-101: URL検証（実際のHTTPリクエスト） 🔵

**信頼性**: 🔵 *ヒアリングQ6: 実際に取得して検証より*

### Given（前提条件）
- ユーザーがURLを入力した状態

### When（実行条件）
- 「登録」ボタンをクリックする

### Then（期待結果）
- サーバーサイドで対象URLへHTTPリクエストが送信される
- レスポンスがRSS/Atom形式として有効か検証される

### テストケース

#### 正常系

- [ ] **TC-101-01**: 有効なRSS URLの検証成功 🔵
  - **入力**: 有効なRSSフィードURL (例: https://example.com/feed.xml)
  - **期待結果**: 検証成功、フィード情報が取得される
  - **信頼性**: 🔵 *ヒアリングQ6より*

#### 異常系

- [ ] **TC-101-E01**: HTTPリクエスト失敗 🔵
  - **入力**: 存在しないURL (例: https://not-exist-example-123.com/feed)
  - **期待結果**: 「URLへの接続に失敗しました」エラーメッセージ表示
  - **信頼性**: 🔵 *ヒアリングQ6より*

- [ ] **TC-101-E02**: RSS形式でないURLの検証失敗 🟡
  - **入力**: HTMLページのURL (例: https://example.com/)
  - **期待結果**: 「有効なRSS/Atomフィードではありません」エラーメッセージ表示
  - **信頼性**: 🟡 *REQ-107から導出*

- [ ] **TC-101-E03**: タイムアウト 🟡
  - **入力**: レスポンスが遅いURL（30秒超）
  - **期待結果**: タイムアウトエラーメッセージ表示
  - **信頼性**: 🟡 *NFR-001から導出*

## REQ-102: フィード情報の自動取得 🔵

**信頼性**: 🔵 *ヒアリングQ1: フィード情報の自動取得より*

### Given（前提条件）
- URL検証が成功した状態

### When（実行条件）
- 登録処理が完了する

### Then（期待結果）
- フィードのタイトルが自動取得され保存される
- フィードの説明が自動取得され保存される（存在する場合）

### テストケース

#### 正常系

- [ ] **TC-102-01**: タイトル・説明の自動取得 🔵
  - **入力**: タイトルと説明を持つRSSフィードのURL
  - **期待結果**: タイトルと説明が正しく取得・保存される
  - **信頼性**: 🔵 *ヒアリングQ1, Q5より*

- [ ] **TC-102-02**: タイトルのみ存在するフィード 🟡
  - **入力**: タイトルはあるが説明のないRSSフィードのURL
  - **期待結果**: タイトルのみ保存、descriptionはnullで保存される
  - **信頼性**: 🟡 *データモデルから推測*

#### 境界値

- [ ] **TC-102-B01**: タイトルが空のフィード 🟡
  - **入力**: タイトルが空のRSSフィードURL
  - **期待結果**: URLをフォールバックとしてタイトルに使用する
  - **信頼性**: 🟡 *EDGE-003から導出*

## REQ-103: 重複URL登録の拒否 🔵

**信頼性**: 🔵 *ヒアリングQ8: 重複登録不可より*

- 同じURLで登録しようとする

### Then（期待結果）
- エラーメッセージが表示される
- フィードは登録されない

### テストケース

#### 異常系

- [ ] **TC-103-E01**: 重複URLの登録拒否 🔵
  - **入力**: 既登録のRSS URL
  - **期待結果**: 「このURLは既に登録されています」エラーメッセージ表示、登録されない
  - **信頼性**: 🔵 *ヒアリングQ8より*

## REQ-104: フィード情報の編集 🔵

**信頼性**: 🔵 *ヒアリングQ1: 編集・削除を選択より*

  - **入力**: 新しいタイトル文字列
  - **期待結果**: タイトルが更新され一覧に反映される
  - **信頼性**: 🔵 *ヒアリングQ1より*

- [ ] **TC-104-02**: メモの追加・編集 🔵
  - **入力**: メモ文字列
  - **期待結果**: メモが保存される
  - **信頼性**: 🔵 *ヒアリングQ5より*

- [ ] **TC-104-03**: メモの削除（空にする） 🟡
  - **入力**: 空文字列のメモ
  - **期待結果**: メモがnullに更新される
  - **信頼性**: 🟡 *データモデルから推測*

## REQ-002/003: フィード一覧表示 🔵

**信頼性**: 🔵 *ヒアリングQ7: 一覧表示項目より*

### Given（前提条件）
- フィード管理ページが表示されている

### When（実行条件）
- ページをロードする

### Then（期待結果）
- 登録済みフィードが一覧表示される
- 各フィードにタイトル・登録日時・最終更新日時が表示される

### テストケース

#### 正常系

- [ ] **TC-002-01**: フィード一覧の表示 🔵
  - **入力**: フィードが3件登録された状態でページアクセス
  - **期待結果**: 3件のフィードが一覧表示される
  - **信頼性**: 🔵 *ヒアリングQ7より*

- [ ] **TC-002-02**: 一覧項目の確認 🔵
  - **入力**: 登録済みフィードの一覧
  - **期待結果**: タイトル・登録日時・最終更新日時が表示される
  - **信頼性**: 🔵 *ヒアリングQ7より*

#### 境界値

- [ ] **TC-002-B01**: フィード0件の表示 🟡
  - **入力**: フィードが0件の状態でページアクセス
  - **期待結果**: 「フィードを登録してください」などの案内メッセージが表示される
  - **信頼性**: 🟡 *REQ-202から導出*
