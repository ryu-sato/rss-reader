# RSSシードの登録 準備タスク（ユーザー作業）

> **仕様**: [requirements.md](requirements.md)
> **生成日**: 2026-03-13

**【信頼性レベル凡例】**:
- 🔵 **青信号**: 要件定義書・設計文書・ユーザヒアリングで明確に必要と判明したタスク
- 🟡 **黄信号**: 要件定義書・設計文書から妥当に推測されるタスク
- 🔴 **赤信号**: 推測による予防的タスク（実装時に不要と判明する可能性あり）

## 必須（実装開始前に完了が必要）

以下のタスクが完了していないと、実装フェーズでブロッカーになります。

- [ ] **Next.jsプロジェクトの初期化** 🔵 *ヒアリングQ4: Next.js選択より*
  - `npx create-next-app@latest rss-reader --typescript --app` などで初期化
  - App Routerを使用する設定にする
  - 関連要件: アーキテクチャ基盤

- [ ] **SQLiteクライアントライブラリの選定・インストール** 🔵 *ヒアリングQ5: SQLite選択より*
  - 選択肢: `better-sqlite3`（同期API）、`Drizzle ORM + SQLite`、`Prisma + SQLite` など
  - 推奨: `Drizzle ORM + better-sqlite3`（TypeScript型安全性が高い）
  - 関連要件: データモデル全般

- [ ] **RSSパーサーライブラリの選定・インストール** 🔵 *REQ-101: RSS/Atom検証より*
  - 選択肢: `rss-parser`、`fast-xml-parser` + 手動パース など
  - サポート形式: RSS 2.0、Atom 1.0
  - 関連要件: REQ-101, REQ-102, REQ-107

## 推奨（実装中に用意できればOK）

実装を開始できますが、該当機能の実装前までに準備してください。

- [ ] **開発用SQLiteデータベースファイルの配置場所の決定** 🟡 *DBセットアップから推測*
  - 例: プロジェクトルートの `db/` ディレクトリ、または `.local/` など
  - `.gitignore` への追加も検討（DB内データをGit管理するか否か）
  - 必要になるフェーズ: DB設計・マイグレーション時
  - 関連要件: データモデル全般

- [ ] **HTTPクライアントライブラリの検討** 🟡 *REQ-101: 実際にフェッチして検証より*
  - Next.jsのサーバーサイドでは `fetch` API（Node.js 18+）が利用可能
  - タイムアウト制御が必要な場合は `AbortController` と組み合わせる
  - 必要になるフェーズ: URL検証実装時
  - 関連要件: REQ-101, NFR-001

## 確認事項（判断が必要）

実装方針に影響するため、早めの判断・確認が推奨されます。

- [ ] **SSRF対策の実施方針** 🔴 *NFR-101: セキュリティ要件（推測）より*
  - プライベートIPアドレス（127.0.0.1、192.168.x.x等）へのリクエストをブロックするか
  - 外部サービス・クラウド環境へのデプロイ予定がある場合は対策を推奨
  - 関連要件: NFR-101（🔴要確認）

- [ ] **エラーメッセージの言語（日本語/英語）の確定** 🔴 *プロジェクト名から推測*
  - 現状: 日本語を想定（requirements.md NFR-202）
  - 確認: 日本語のみか、i18n対応が必要か
  - 関連要件: NFR-202（🔴要確認）

- [ ] **フィード説明・メモの最大文字数** 🔴 *データモデル設計から推測*
  - フィード説明（description）の最大長（EDGE-004）
  - ユーザーメモ（memo）の最大長（EDGE-102）
  - DB設計に影響するため早めに決定推奨
  - 関連要件: EDGE-004, EDGE-102（🔴要確認）
