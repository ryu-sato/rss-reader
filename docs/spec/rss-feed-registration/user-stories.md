## エピック1: RSSフィード登録

### ストーリー 1.1: URLを入力してフィードを登録する 🔵

**信頼性**: 🔵 *ヒアリングQ1: URLを入力して登録、フィード情報の自動取得より*

**私は** アプリの利用者 **として**
**RSSフィードのURLを入力して登録したい**
**そうすることで** 好きなサイトの新着記事を購読できる

**関連要件**: REQ-001, REQ-101, REQ-102, REQ-103, REQ-106, REQ-107

**詳細シナリオ**:
1. フィード登録フォームにRSSフィードのURLを入力する
2. 「登録」ボタンをクリックする
3. システムがサーバーサイドで対象URLへHTTPリクエストを送信する
4. レスポンスが有効なRSS/Atom形式であることを確認する
5. フィードのタイトルと説明を自動取得する
6. フィード情報をDBに保存する
7. 登録完了のフィードバックを表示し、一覧に追加される

**前提条件**:
- アプリにアクセスできる状態
- URLがhttps://またはhttp://から始まる

**制約事項**:
- 同一URLは重複登録不可
- URL長は2048文字以内

**優先度**: Must Have

---

### ストーリー 1.2: URL検証中のフィードバックを受け取る 🟡

**信頼性**: 🟡 *UX要件として妥当な推測*

**私は** アプリの利用者 **として**
**URL検証中にローディング表示を見たい**
**そうすることで** 処理中であることを把握できる

**関連要件**: REQ-201

**詳細シナリオ**:
1. URLを入力して登録ボタンをクリックする
2. ボタンが非活性化またはスピナーが表示される
3. 検証完了まで待機する
4. 成功または失敗のメッセージが表示される

**前提条件**:
- 登録フォームを開いている

**優先度**: Should Have

---

### ストーリー 1.3: 無効なURLで登録しようとしたときエラーを受け取る 🔵

**信頼性**: 🔵 *ヒアリングQ6: 実際に取得して検証、Q7: 重複登録不可より*

**私は** アプリの利用者 **として**
**無効なURLや既登録URLを入力したときに明確なエラーを受け取りたい**
**そうすることで** 何が問題か理解して修正できる

**関連要件**: REQ-103, REQ-106, REQ-107, REQ-402

**詳細シナリオ**:
1. 無効なURLや既登録URLを入力して登録ボタンをクリックする
2. システムが適切なエラーメッセージを表示する
   - 重複URL: 「このURLは既に登録されています」
   - 取得失敗: 「URLへの接続に失敗しました」
   - RSS形式エラー: 「有効なRSS/Atomフィードではありません」

**前提条件**:
- 登録フォームを開いている

**優先度**: Must Have

## エピック2: フィード一覧表示

### ストーリー 2.1: 登録済みフィード一覧を確認する 🔵

**信頼性**: 🔵 *ヒアリングQ7: 一覧表示項目より*

**私は** アプリの利用者 **として**
**登録済みRSSフィードの一覧を見たい**
**そうすることで** どのフィードを購読しているか把握できる

**関連要件**: REQ-002, REQ-003

**詳細シナリオ**:
1. アプリのフィード管理ページを開く
2. 登録済みフィードの一覧が表示される
3. 各フィードにタイトル・登録日時・最終更新日時が表示される

**前提条件**:
- 少なくとも1件のフィードが登録済み

**優先度**: Must Have

---

### ストーリー 2.2: フィードが未登録の場合に登録を促すメッセージを見る 🟡

**信頼性**: 🟡 *UX要件として妥当な推測*

**私は** 新しく利用を開始したアプリの利用者 **として**
**フィードがない場合に登録方法を案内してほしい**
**そうすることで** 迷わず最初のフィードを登録できる

**関連要件**: REQ-202

**詳細シナリオ**:
1. フィード管理ページを開く
2. フィードが1件も登録されていない場合
3. 「フィードを登録してください」等の案内メッセージが表示される
4. フィード登録フォームへのリンクまたはボタンが提示される

**前提条件**:
- フィードが0件の状態

**優先度**: Should Have

---

## エピック3: フィード編集

### ストーリー 3.1: 登録済みフィードのメモを編集する 🔵

**信頼性**: 🔵 *ヒアリングQ1: フィードの編集・削除を選択、Q5: メモ保存より*

**私は** アプリの利用者 **として**
**登録済みフィードのメモを編集したい**
**そうすることで** フィードに関する覚え書きを残せる

**関連要件**: REQ-104

**詳細シナリオ**:
1. フィード一覧から編集したいフィードを選択する
2. 編集フォームが表示される
3. タイトル・説明・メモを変更する
4. 「保存」ボタンをクリックする
5. 変更が保存され一覧に反映される

**前提条件**:
- 少なくとも1件のフィードが登録済み

**優先度**: Must Have

---

## エピック4: フィード削除

### ストーリー 4.1: 不要なフィードを削除する 🔵

**信頼性**: 🔵 *ヒアリングQ9: 確認ダイアログあり、Q10: 優先度Must Haveより*

**私は** アプリの利用者 **として**
**不要になったRSSフィードを削除したい**
**そうすることで** 購読リストをすっきりと管理できる

**関連要件**: REQ-105

**詳細シナリオ**:
1. フィード一覧から削除したいフィードの削除ボタンをクリックする
2. 確認ダイアログが表示される（「本当に削除しますか？」）
3. 「削除する」ボタンをクリックすると削除実行される
4. 「キャンセル」ボタンをクリックすると削除キャンセルされる
5. 削除後、一覧からそのフィードが消える

**前提条件**:
- 少なくとも1件のフィードが登録済み

**制約事項**:
- 削除前に必ず確認ダイアログを表示する

**優先度**: Must Have
