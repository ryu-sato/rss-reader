## ユーザーストーリー一覧

### ストーリー1: フィード購読

**として** 情報収集をしたいユーザー
**私は** RSSフィードのURLを登録してフィードを購読
**をしたい**
**そうすることで** 複数サイトの最新記事を一か所で確認できる

**実装根拠**:
- [feed-form.tsx](../../../src/components/feed-form.tsx) - URL入力フォーム
- [POST /api/feeds](../../../src/app/api/feeds/route.ts) - フィード登録API
- [feed-service.ts](../../../src/lib/feed-service.ts) - フィード作成ロジック

**受け入れ基準**:
- URLを入力して送信するとフィードが登録される
- 重複URLは登録を拒否し、エラーメッセージを表示する
- 無効なURL、プライベートIPは登録を拒否する
- 登録時にタイトル・説明・ファビコンが自動取得される

**として** フィードを整理したいユーザー

**をしたい**
**そうすることで** フィード一覧を自分の用途に合わせて整理できる

**実装根拠**:
- [feed-list.tsx](../../../src/components/feed-list.tsx) - フィード一覧テーブル
- [edit-feed-form.tsx](../../../src/components/edit-feed-form.tsx) - 編集フォーム
- [PUT/DELETE /api/feeds/[id]](../../../src/app/api/feeds/[id]/route.ts)

**受け入れ基準**:
- フィード管理ページでタイトル・説明・メモを編集できる
- フィードを削除すると関連記事も削除される
- 削除前に確認ダイアログが表示される

### ストーリー3: 記事閲覧

**として** ニュースを効率よく読みたいユーザー
**私は** 購読フィードの最新記事をカードグリッドで閲覧し、記事をモーダルで読む
**をしたい**
**そうすることで** 複数フィードの記事を一元的に、素早く確認できる

**実装根拠**:
- [entry-card-grid.tsx](../../../src/components/entry-card-grid.tsx) - カードグリッド
- [entry-card.tsx](../../../src/components/entry-card.tsx) - 記事カード
- [article-modal.tsx](../../../src/components/article-modal.tsx) - 記事モーダル

**受け入れ基準**:
- 記事はサムネイル・タイトル・要約・日付付きカードで表示される
- カードをクリックするとモーダルで記事内容が表示される
- モーダル表示時に自動的に既読マークが付く
- スクロールで追加記事が自動ロードされる（無限スクロール）

### ストーリー4: 記事フィルタリング

**として** 特定のフィードや話題に集中したいユーザー
**私は** フィード別・タグ別・キーワードで記事をフィルタリング
**をしたい**
**そうすることで** 関心のある記事だけを効率的に読める

**実装根拠**:
- [entry-filter.tsx](../../../src/components/entry-filter.tsx) - フィルタUI
- [entry-filter-bar.tsx](../../../src/components/entry-filter-bar.tsx) - フィルタバー
- [read-filter.tsx](../../../src/components/read-filter.tsx) - 既読/未読切り替え

**受け入れ基準**:

- 未読/全記事の切り替えができる

### ストーリー5: 既読管理

**として** 読んだ記事と未読記事を管理したいユーザー
**私は** 記事の既読/未読状態を管理し、未読記事のみを素早く確認
**をしたい**
**そうすることで** 何を読んだか把握でき、未読記事を見逃さない

**実装根拠**:
- [article-modal.tsx](../../../src/components/article-modal.tsx) - 既読トグル
- [entry-card.tsx](../../../src/components/entry-card.tsx) - カードの既読アイコン
- [entry-service.ts](../../../src/lib/entry-service.ts) - updateEntryMeta

**受け入れ基準**:
- 記事を開くと自動的に既読になる
- 記事カードとモーダルから手動で既読/未読を切り替えられる
- 同一リンクの記事を既読にすると、他のフィードの同記事も既読になる
- サイドバーの未読バッジがリアルタイムに更新される

### ストーリー10: フィード自動更新

**として** 常に最新の記事を読みたいユーザー
**私は** バックグラウンドで自動的にフィードが更新される
**ことを期待する**
**そうすることで** 手動更新不要で常に最新記事を確認できる

**実装根拠**:
- [cron.ts](../../../src/lib/cron.ts) - 毎時自動更新
- [fetchAllFeedsEntries()](../../../src/lib/entry-service.ts) - 全フィード更新

**受け入れ基準**:
- サーバー起動後、毎時0分に全フィードが自動更新される

- サイドバーのリフレッシュボタンで手動更新もできる
