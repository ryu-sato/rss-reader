## 実装詳細

Next.jsの開発環境でのホットリロード時にPrismaクライアントが複数インスタンス生成される問題を防ぐシングルトン実装。

```typescript
// グローバルオブジェクトにキャッシュ（開発環境のホットリロード対策）
const globalForPrisma = globalThis as unknown as { prisma: PrismaClient }

export const prisma =
  globalForPrisma.prisma ||
  new PrismaClient({
    adapter: new PrismaLibSQL(createClient({ url: process.env.DATABASE_URL! }))
  })

if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma
```

### 使用ライブラリ
- `@prisma/client`: Prisma ORM本体
- `@prisma/adapter-libsql`: LibSQL（SQLite互換）アダプター
- `@libsql/client`: LibSQLクライアント
