## 実装詳細

Next.js App RouterのRoute Handlersを使ったREST API実装。

### GET /api/feeds
- `feed-service.getAllFeeds()` を呼び出し
- レスポンス: `{ success: true, data: FeedListItem[] }`

### POST /api/feeds
- リクエストボディ: `{ url: string }`
- バリデーション: urlフィールドが存在し文字列であること
- `feed-service.createFeed(url)` を呼び出し
- レスポンス: `{ success: true, data: Feed }` (HTTP 201)

### GET /api/feeds/[id]
- `feed-service.getFeedById(id)` を呼び出し
- レスポンス: `{ success: true, data: Feed }`

### PUT /api/feeds/[id]
- リクエストボディ: `{ title?, description?, memo? }`
- バリデーション: titleが空文字でないこと
- `feed-service.updateFeed(id, data)` を呼び出し
- レスポンス: `{ success: true, data: Feed }`

### DELETE /api/feeds/[id]
- `feed-service.deleteFeed(id)` を呼び出し
- レスポンス: `{ success: true }`

### 共通エラーハンドリング
- AppErrorインスタンスを`{ success: false, error: { code, message } }`にJSONシリアライズ
- statusCodeをHTTPレスポンスコードに設定
- 予期しないエラーは500で返却
