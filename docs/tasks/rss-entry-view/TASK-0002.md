## 実装詳細

### 1. エンティティ型定義 🔵

**信頼性**: 🔵 *設計文書 interfaces.ts エンティティ定義より*

**実装ファイル**: `src/types/entry.ts`

- `Entry`: id, feedId, guid, title, link, description, content, publishedAt, createdAt, updatedAt
- `EntryListItem`: 一覧表示用軽量版（feed, meta, tags含む）
- `EntryDetail extends Entry`: モーダル詳細用（feed, meta, tags含む）
- `EntryMeta`: id, entryId, isRead, isReadLater, createdAt, updatedAt
- `Tag`: id, name, createdAt
- `TagOnEntry`: entryId, tagId, tag

### 2. APIリクエスト/レスポンス型 🔵

**信頼性**: 🔵 *設計文書 interfaces.ts APIリクエスト/レスポンスより*

- `GetEntriesQuery`: feedId?, tagId?, page?, limit?, afterId?, beforeId?
- `GetEntriesResponse`: success, data: EntryListItem[], pagination
- `GetEntryResponse`: success, data: EntryDetail
- `UpdateEntryMetaRequest`: isRead?, isReadLater?
- `UpdateEntryMetaResponse`: success, data: EntryMeta
- `CreateTagRequest`: name, entryId
- `CreateTagResponse`: success, data: Tag
- `GetTagsResponse`: success, data: Tag[]

### 3. エラーコード型 🔵

**信頼性**: 🔵 *既存 src/types/feed.ts ErrorCode パターンより*

- `EntryErrorCode`: 'ENTRY_NOT_FOUND' | 'TAG_NOT_FOUND' | 'VALIDATION_ERROR' | 'INTERNAL_SERVER_ERROR'

### 4. 内部型（フェッチャー・Prisma入力） 🔵

**信頼性**: 🔵 *設計文書 interfaces.ts より*

- `FetchedEntryData`: guid, title, link, description, content, publishedAt
- `CreateEntryInput`: Omit<Entry, 'id' | 'createdAt' | 'updatedAt'>
- `UpdateEntryMetaInput`: Partial<Pick<EntryMeta, 'isRead' | 'isReadLater'>>
