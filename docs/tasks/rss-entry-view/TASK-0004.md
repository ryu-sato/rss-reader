## 実装詳細

### 1. fetchEntries 関数実装 🔵

**信頼性**: 🔵 *設計文書 architecture.md EntryFetcher層より*

**実装ファイル**: `src/lib/entry-fetcher.ts`

```typescript
import Parser from 'rss-parser'
import { FetchedEntryData } from '@/types/entry'

const parser = new Parser({
  customFields: {
    item: [['content:encoded', 'contentEncoded']]
  }
})

export async function fetchEntries(feedUrl: string): Promise<FetchedEntryData[]> {
  const feed = await parser.parseURL(feedUrl)
  return feed.items.map(item => ({
    guid: item.guid ?? item.link ?? item.title ?? '',
    title: item.title ?? item.link ?? '',
    link: item.link ?? '',
    description: item.summary ? stripHtml(item.summary) : null,
    content: item.contentEncoded
      ? stripHtml(item.contentEncoded)
      : item.content
        ? stripHtml(item.content)
        : null,
    publishedAt: item.pubDate ? new Date(item.pubDate) : null,
  }))
}

function stripHtml(html: string): string {
  return html.replace(/<[^>]*>/g, '').replace(/&amp;/g, '&').replace(/&lt;/g, '<').replace(/&gt;/g, '>').replace(/&quot;/g, '"').replace(/&#39;/g, "'").trim()
}
```

### 2. SSRF対策との連携 🔵

**信頼性**: 🔵 *設計文書 NFR-102・既存ssrf-guard.tsより*

EntryFetcher を呼び出す EntryService 側で `ssrf-guard.ts` を使用して URL 検証を行う（entry-fetcher.ts 内では行わない）。

### 3. エッジケース対応 🟡

**信頼性**: 🟡 *要件定義 EDGE-001, EDGE-002, EDGE-003から妥当な推測*

- content がない場合は description を使用（EDGE-001）
- publishedAt がない場合は null を返す（EDGE-002: 呼び出し側で処理）
- title が空の場合は link をフォールバックに使用（EDGE-003）
