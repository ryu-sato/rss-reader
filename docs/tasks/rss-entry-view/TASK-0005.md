## 実装詳細

### 1. saveEntries 関数（重複排除・upsert） 🔵

**信頼性**: 🔵 *要件定義 REQ-103・設計文書 EntryService層より*

**実装ファイル**: `src/lib/entry-service.ts`

```typescript
export async function saveEntries(feedId: string, entries: FetchedEntryData[]): Promise<void> {
  for (const entry of entries) {
    await prisma.entry.upsert({
      where: { feedId_guid: { feedId, guid: entry.guid } },
      create: {
        feedId,
        guid: entry.guid,
        title: entry.title,
        link: entry.link,
        description: entry.description,
        content: entry.content,
        publishedAt: entry.publishedAt,
      },
      update: {
        title: entry.title,
        link: entry.link,
        description: entry.description,
        content: entry.content,
        publishedAt: entry.publishedAt,
      },
    })
  }
}
```

### 2. 500件上限チェック・古いエントリー削除 🔵

**信頼性**: 🔵 *要件定義 REQ-102, REQ-401より*

```typescript
const MAX_ENTRIES_PER_FEED = 500

export async function enforceEntryLimit(feedId: string): Promise<void> {
  const count = await prisma.entry.count({ where: { feedId } })
  if (count > MAX_ENTRIES_PER_FEED) {
    const excess = count - MAX_ENTRIES_PER_FEED
    const oldestEntries = await prisma.entry.findMany({
      where: { feedId },
      orderBy: [{ publishedAt: 'asc' }, { createdAt: 'asc' }],
      take: excess,
      select: { id: true },
    })
    await prisma.entry.deleteMany({
      where: { id: { in: oldestEntries.map(e => e.id) } },
    })
  }
}
```

### 3. fetchAllFeedsEntries（定期取得メイン処理） 🔵

**信頼性**: 🔵 *要件定義 REQ-211, REQ-104, EDGE-004・データフロー図より*

```typescript
export async function fetchAllFeedsEntries(): Promise<void> {
  const feeds = await prisma.feed.findMany()
  for (const feed of feeds) {
    try {
      await validateUrl(feed.url) // ssrf-guard
      const entries = await fetchEntries(feed.url)
      await saveEntries(feed.id, entries)
      await enforceEntryLimit(feed.id)
      await prisma.feed.update({
        where: { id: feed.id },
        data: { lastFetchedAt: new Date() },
      })
    } catch (error) {
      console.error(`Failed to fetch entries for feed ${feed.id}: ${error}`)
      // 次のフィードを継続（REQ-104, EDGE-004）
    }
  }
}
```

## 単体テスト要件

### テストケース1: saveEntries - 重複スキップ 🔵

**信頼性**: 🔵 *REQ-103より*

**Given**: 同じ feedId + guid のエントリーが既にDBに存在する
**When**: `saveEntries(feedId, [同じguidのentry])` を呼び出す
**Then**: DBのエントリー数が変わらない（重複登録されない）

### テストケース2: enforceEntryLimit - 500件超過時削除 🔵

**信頼性**: 🔵 *REQ-102, REQ-401より*

**Given**: あるフィードに501件のエントリーが存在する
**When**: `enforceEntryLimit(feedId)` を呼び出す
**Then**: 最古の1件が削除され500件になる

### テストケース3: fetchAllFeedsEntries - 個別失敗時継続 🟡

**信頼性**: 🟡 *REQ-104, EDGE-004から妥当な推測*

**Given**: 2つのフィードがあり、1つ目の取得が失敗する
**When**: `fetchAllFeedsEntries()` を呼び出す
**Then**: 2つ目のフィードの処理が実行される（エラーログが出る）
