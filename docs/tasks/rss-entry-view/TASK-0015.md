## 実装詳細

### 1. モーダル表示・閉じる 🔵

**信頼性**: 🔵 *設計文書 URLクエリパラメータ・データフロー図より*

**実装ファイル**: `src/components/entry-modal.tsx`

```typescript
'use client'
import { useRouter, useSearchParams } from 'next/navigation'
import { useEffect, useState } from 'react'
import { EntryDetail } from '@/types/entry'

interface EntryModalProps {
  entry: EntryDetail
}

export function EntryModal({ entry }: EntryModalProps) {
  const router = useRouter()
  const searchParams = useSearchParams()

  const closeModal = () => {
    const params = new URLSearchParams(searchParams.toString())
    params.delete('entryId')
    router.push(`/?${params.toString()}`)
  }

  // 自動既読（REQ-101）
  useEffect(() => {
    if (!entry.meta?.isRead) {
      fetch(`/api/entries/${entry.id}/meta`, {
        method: 'PUT',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ isRead: true }),
      })
    }
  }, [entry.id])

  return (
    <dialog open>
      <button onClick={closeModal}>×</button>
      <h2>{entry.title}</h2>
      <time>{entry.publishedAt?.toLocaleString()}</time>
      <a href={entry.link} target="_blank" rel="noopener noreferrer">元記事を読む</a>
      <p>{entry.content ?? entry.description}</p>
      {/* 前後ナビ・既読・後で読む・タグ入力は別セクション */}
    </dialog>
  )
}
```

### 2. 前後ナビゲーション 🔵

**信頼性**: 🔵 *REQ-007・データフロー図 機能3より*

- 「前の記事」: `GET /api/entries?beforeId=currentId` で前のエントリーIDを取得して URL 更新
- 「次の記事」: `GET /api/entries?afterId=currentId` で次のエントリーIDを取得して URL 更新
- 最初/最後のエントリーではボタンを disabled

### 3. 既読・後で読む操作 🔵

**信頼性**: 🔵 *REQ-008, REQ-009, NFR-201・データフロー図 機能4より*

- ボタンクリックで楽観的更新 → `PUT /api/entries/:id/meta` を呼び出す
- API失敗時は楽観的更新をロールバック

### 4. 自動既読マーク 🔵

**信頼性**: 🔵 *REQ-101より（条件付き要件: モーダル開時に既読）*

- `useEffect` でモーダル表示時に `PUT /api/entries/:id/meta { isRead: true }` を呼び出す
- 既に既読の場合はスキップ
