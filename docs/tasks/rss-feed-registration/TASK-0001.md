## 実装詳細

### 1. Next.jsプロジェクト初期化 🔵

**信頼性**: 🔵 *note.md 技術スタック・devcontainer.jsonより*

```bash
npx create-next-app@latest . \
  --typescript \
  --tailwind \
  --eslint \
  --app \
  --src-dir=false \
  --import-alias="@/*"
```

**設定確認項目**:
- `tsconfig.json`: `"strict": true` が設定されている
- `tailwind.config.ts`: content パスが設定されている
- `app/` ディレクトリ構造が生成されている

### 2. 依存パッケージのインストール 🔵

**信頼性**: 🔵 *architecture.md コンポーネント構成・design-interview.mdより*

```bash
# ORM
npm install @prisma/client
npm install --save-dev prisma

# RSSパーサー
npm install rss-parser
npm install --save-dev @types/rss-parser

# テストフレームワーク
npm install --save-dev vitest @vitejs/plugin-react jsdom @testing-library/react @testing-library/jest-dom @testing-library/user-event
```

### 3. shadcn/ui のセットアップ 🔵

**信頼性**: 🔵 *design-interview.md Q4: shadcn/ui + Tailwind CSより*

```bash
npx shadcn@latest init
```

必要なコンポーネントを追加:
```bash
npx shadcn@latest add button
npx shadcn@latest add dialog
npx shadcn@latest add alert-dialog
npx shadcn@latest add form
npx shadcn@latest add input
npx shadcn@latest add textarea
npx shadcn@latest add label
npx shadcn@latest add toast
```

### 4. Vitest 設定ファイル作成 🔵

**信頼性**: 🔵 *design-interview.md Q1: Vitestより*

**実装ファイル**: `vitest.config.ts`

```typescript
import { defineConfig } from 'vitest/config'
import react from '@vitejs/plugin-react'
import path from 'path'

export default defineConfig({
  plugins: [react()],
  test: {
    environment: 'jsdom',
    globals: true,
    setupFiles: ['./vitest.setup.ts'],
  },
  resolve: {
    alias: {
      '@': path.resolve(__dirname, '.'),
    },
  },
})
```

**実装ファイル**: `vitest.setup.ts`

```typescript
import '@testing-library/jest-dom'
```

### 5. package.json スクリプト追加 🔵

**信頼性**: 🔵 *テスト実行環境の標準設定より*

`package.json` の scripts に追加:
```json
{
  "scripts": {
    "test": "vitest",
    "test:run": "vitest run",
    "test:coverage": "vitest run --coverage"
  }
}
```
