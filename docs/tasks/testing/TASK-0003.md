## 実装詳細

### rss-fetcher.test.ts
モック: `fetch` API, `rss-parser`

テストケース:
- RSS 2.0フィードの正常パース
- Atom 1.0フィードの正常パース
- ネットワークエラー時の FeedFetchError
- タイムアウト（30秒）時の FeedFetchError
- HTTP非200レスポンス時の FeedFetchError
- 無効なRSSフォーマット時の InvalidFeedFormatError
- titleが空の場合にURLをフォールバックとして使用
- descriptionが未設定の場合の処理

### ssrf-guard.test.ts
モック: `dns.promises.lookup`

テストケース:
- 全プライベートIPv4レンジのブロック確認
  - 127.x.x.x (ループバック)
  - 10.x.x.x
  - 172.16-31.x.x
  - 192.168.x.x
  - 169.254.x.x (リンクローカル)
  - 100.64-127.x.x (CGNAT)
- IPv6プライベートアドレスのブロック
- パブリックIPの許可確認
- http/https以外のプロトコルのブロック
- 2048文字超えURLのブロック
- 不正URLフォーマットのブロック
- ホスト名のDNS解決と結果IPのチェック

### errors.test.ts
テストケース（各エラークラスの検証）:
- ConflictError: instanceof AppError, code='FEED_ALREADY_EXISTS', statusCode=409
- NotFoundError: instanceof AppError, code='FEED_NOT_FOUND', statusCode=404
- FeedFetchError: instanceof AppError, code='FEED_FETCH_FAILED', statusCode=422
- InvalidFeedFormatError: instanceof AppError, code='INVALID_FEED_FORMAT', statusCode=422
- SSRFError: instanceof AppError, code='URL_NOT_ALLOWED', statusCode=400
